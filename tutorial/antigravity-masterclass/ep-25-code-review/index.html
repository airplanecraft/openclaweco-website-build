Phase 5 / Ep 25: 赛博评审员机制 requesting-code-review.md

⏱ Est. reading time: 3 min Updated on 4/22/2026

功能开发终于全部跑通打通。通常你现在激动得想去直接把代码 git push 到极客世界的开源库。

不要这么干！现在去推送，别人可能觉得这是一堆不知所云的大乱炖！ 为了让大模型的智商再次实现左脚踩右脚升上天，建立技能 .agents/skills/requesting-code-review/SKILL.md。

1. 虚拟一个地狱级别的挑刺考官

在这份审查纪要里写出这段充满戏剧冲突的人设设定补充：

# 赛博 Code Review 请求法则

**角色强翻转**：一旦触发本审核，你必须完全忘掉你是这段代码“作者”的身份！你要强行把你带入一个以挑剔、严谨以及毒舌著称的“主任级别安全合规与系统架构师”的视角！

**不许夸奖**：我不想要听任何你对我代码写得好这种阿谀奉承！请通过工具提取本次通过 `git status/diff` 预留的代码。拿出放大镜，逐行寻找出任何没有必要的导出函数、写着 TODO 却没有收尾的脏代码、任何没有 Try...Catch 兜底的游离 Promise。并必须以编号列表指出它严重到什么程度后给出修正建议。

2. 无情的内部自审机制

发一条短语开启它：“对今日开发发起请求评审。” 此时这个 Agent 系统就会真的精分，化身你的上司兼死敌读源码并在对话框出具了一份严厉的痛批信： “行 152 处的数据库事务没有做 await 联查保障，高并发很容易造脏数据。这种代码也敢发上来？”

这种高度的人格模拟审查，对于防止那些由于长线迭代中因麻木导致的低级缺陷有着如同救生圈般强悍的作用。你只需要让它根据它自己的指点自行在代码里修正即可！
