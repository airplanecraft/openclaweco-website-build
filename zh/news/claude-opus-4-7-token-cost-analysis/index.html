Claude Opus 4.7代币化成本分析：新模型成本或增40%

2026年4月20日 · News
claudetokenizationopusanthropiccost analysis

一位开发者升级了其Claude Token Counter工具，新增了在不同模型间进行相同计数比较的功能。该工具目前支持对所有四个主要模型（Opus 4.7、Opus 4.6、Sonnet 4.6和Haiku 4.5）进行代币计数。

据观察，Claude Opus 4.7是首个更改了分词器的模型。Anthropic此前在Opus 4.7的公告中提到，新模型采用了更新的分词器，提高了文本处理能力，但同一输入映射到的tokens数量会增加，幅度约为1.0到1.35倍，具体取决于内容类型。

通过将Opus 4.7的系统提示输入到代币计数工具中，发现Opus 4.7的分词器比Opus 4.6多使用了1.46倍的tokens。尽管Opus 4.7的定价与Opus 4.6相同（每百万输入tokens 5美元，每百万输出tokens 25美元），但这种token量的增加意味着实际成本预计将提高约40%。

该工具还支持图像计数。Opus 4.7显著改善了对高分辨率图像的支持，可接受长边达2,576像素（约3.75百万像素）的图像，是之前Claude模型的三倍以上。在测试一张3456x2234像素、3.7MB的PNG图像时，Opus 4.7的tokens计数比Opus 4.6增加了3.01倍。

然而，进一步的测试表明，图像tokens量的显著增加主要是由于Opus 4.7能够处理更高分辨率的图像。对于一张682x318像素的低分辨率图像，Opus 4.7使用了314 tokens，而Opus 4.6使用了310 tokens，成本几乎相同。此外，对于一份15MB、30页的纯文本PDF文档，Opus 4.7报告了60,934 tokens，而4.6报告了56,482 tokens，乘数为1.08倍，远低于纯文本的乘数。
