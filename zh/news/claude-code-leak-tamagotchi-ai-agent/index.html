Anthropic Claude代码意外泄露，“电子宠物”与秘密AI Agent浮出水面

2026年4月2日 · News
claudeanthropicai agentsource codetamagotchi

Anthropic旗下明星聊天机器人Claude的源代码近日被意外泄露，引发了网络社区的广泛关注。数以万计的代码行被网友们迅速“挖掘”，并从中发现了许多令人好奇的功能。

在Reddit的r/ClaudeAI子版块上，一位用户宣称在泄露代码中找到了一个类似“电子宠物（Tamagotchi）”的系统。这个名为“/buddy”的完整宠物系统，允许用户根据自己的ID孵化出一个独特的ASCII字符伴侣。这个虚拟宠物会显示在输入框旁边，并根据用户的编码活动做出反应。

该用户表示，他们发现了18种不同的宠物，包括鸭子、龙、水豚以及一种名为“chonk”的生物，并且这个系统还带有类似于“扭蛋（Gacha）”游戏的稀有度机制，通过随机分配给用户宠物。不过，代码中出现的字符串“friend-2026-401”强烈暗示，这很可能只是Anthropic为2026年愚人节准备的一次性彩蛋。

除了“电子宠物”，互联网侦探们还发现了代号为“Kairos”的更重要功能。据称，Kairos是一个始终在线的AI Agent，能在后台持续运行，并且无需用户明确指令即可代为执行操作。查看代码的用户表示，它甚至可以向手机或桌面发送推送通知以提醒用户。

此外，还有人发现了一个“卧底（undercover）”模式，用于在Claude向公共代码仓库贡献代码时隐藏其AI身份。另一个发现是心情追踪功能，它能根据程序员的消息和如脏话等线索，衡量他们的“挫败感”水平。甚至有用户挖出Anthropic一位程序员留下的评论，其中写道：“这里的记忆化（memoization）大大增加了复杂性，我不确定它是否真的提高了性能。”

总的来说，这次泄露虽然没有发现所谓的“确凿证据”，但无疑为外界提供了一个窥探Anthropic技术内部运作的独特视角，也为竞争对手逆向工程其技术提供了便利。

对Anthropic而言，这无疑是一次令人尴尬的失误。据《The Register》解释，此次代码泄露似乎是由于该公司在公开版本2.1.88的Claude Code npm软件包中意外遗留了一个源映射文件（source map file）所致。源映射文件能将捆绑的代码链接回原始源代码，一位机智的程序员正是利用它找到了Claude的源代码存储位置，并将其完整备份到了GitHub上。

Anthropic已紧急采取行动，通过发出版权下架通知来撤回暴露的源代码，但此时代码可能已在互联网上广泛传播。
