OpenAI GPT-5.5集成Databricks，通过Unity AI网关全面治理

2026年4月25日 · News
gpt-5.5databricksunity ai gatewayai agentsllm governance

OpenAI 最前沿的强大模型 GPT-5.5，专为企业级智能代理工作、复杂文档推理和长周期编码代理任务设计，现已在 Databricks 上获得原生支持。这一集成使得企业客户能够将 GPT-5.5 应用于其专有数据，并实现端到端的治理。

客户可以利用 GPT-5.5 增强编码工作流（无论是通过 OpenAI 的 Codex 还是其他编码代理），构建基于企业数据的复杂代理，为文档处理管道注入更深层的智能，并将数据洞察无缝融入员工的日常工作中。

Databricks 上所有的 GPT-5.5 使用都将通过 Unity AI Gateway 进行全面治理。该网关从一开始就为客户提供了集中化的安全性、精细的成本控制和强大的可观测性，覆盖所有代理、查询和编码工作流。

Unity AI Gateway 作为 GPT-5.5 模型推理和 Codex 编码工作流的统一治理层，通过单一控制平面进行管理。它为 GPT-5.5 引入了企业级控制功能，具体包括：

  • 权限与速率限制：根据用户和组控制访问权限，有效防止成本失控。
  • 安全护栏：检测个人身份信息 (PII)，阻止提示注入攻击，并强制执行内容安全策略，所有这些都可根据端点进行配置。
  • MCP 治理：对每一次代理工具调用进行审计，提供完整的可追溯性。
  • 自动故障转移：当达到速率限制时，流量将自动路由至备用模型。
  • 可观测性：每次请求，无论是模型调用还是 Codex 交互，其身份、令牌、延迟和成本数据都将被记录在客户拥有的 Delta 表中。
  • 统一账单：整合 GPT-5.5、Codex 以及所有其他模型/编码工具的账单。

此外，Databricks 还支持用户在完整的企业上下文中部署 GPT-5.5。平台中的自然语言分析体验 Genie，由包括 GPT-5.5 在内的多个前沿大型语言模型驱动，允许业务用户通过简单的英语安全地与复杂的企业数据交互。这极大地简化了数据探索、即席问题解答以及自动化以往耗时的工作。用户还可以利用 Agent Bricks 构建自定义代理，充分发挥 GPT-5.5 增强的执行和推理能力。
