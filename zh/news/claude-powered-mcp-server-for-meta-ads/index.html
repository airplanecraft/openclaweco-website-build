Claude驱动的AI智能体：开源MCP服务器助Meta广告管理效率飞跃

2026年3月28日 · News
claudemeta adsai agentmarketing apimcp

一位数字营销专家在长期手动通过Meta Marketing API管理37个客户广告账户后，开发并开源了一个创新的Meta Ads MCP（Meta Conversion Platform）服务器。该服务器允许用户直接通过Claude代码实现Meta广告的全方位管理，极大地提升了操作效率。

该名为“KonQuest Meta Ads MCP”的开源项目（MIT许可）提供了57项核心工具，专为通过Claude代码管理Meta广告设计，涵盖以下关键功能：

  • 广告系列管理：支持创建、读取、更新广告系列、广告组、广告和创意。用户甚至可以通过一条指令复制包含所有广告的整个广告系列。
  • 多素材广告：利用asset_feed_spec创建包含视频（9:16 + 1:1）和静态图片（1:1 + 4:5 + 9:16）的单一广告，避免为不同格式重复创建广告。
  • 定位工具包：搜索兴趣、行为、人口统计信息和地理位置。在创建广告组之前，可以预估受众规模。
  • 像素诊断：检查像素健康状态，验证事件触发情况，测试事件发送，并诊断任何URL上的跟踪问题。
  • 目录与DPA：管理产品目录、数据源和产品集，用于动态产品广告，并可带过滤器创建和更新产品集。
  • 设置检查器：为每个账户提供超过42项自动化检查，包括Token健康、像素状态、主页连接、Instagram身份验证和目录链接。

实际操作流程

通过Claude代码管理Meta广告的流程高效且智能：

  • 当用户指示“为我的护肤品商店创建针对25-45岁对美容感兴趣的女性的转化广告系列”时，Claude会读取用户在“库”（vault）中存储的品牌声音、理想客户画像（ICP）和营销角度等文件，验证所有信息，强制执行命名规范，检查像素是否正常触发，并自动创建：
    • 已暂停的广告系列
    • 包含目标设定的广告组
    • 带有创意的广告
    所有广告在得到用户明确批准之前都不会上线。
  • 当用户询问“C5广告系列本周的ROAS是多少？”时，Claude会提取洞察数据，按广告组进行细分，标记表现不佳的广告，并建议预算调整。
  • 当用户需要“复制C3广告系列，但定位更改为30-50岁的男性”时，Claude会复制完整的结构（广告系列+广告组+广告），应用新的定位，并保持命名规范一致。

安全至上

该服务器将安全置于首位，所有写入操作都受到严格监督：

  • 广告默认暂停：所有创建的广告都处于“已暂停”状态，未经用户明确批准不会上线。
  • 命名强制执行：在API调用发生之前，所有对象都必须遵循预设的命名规范。
  • 验证流程：在每次写入之前，都会运行合规性、创意规格、跟踪、结构和操作检查。
  • 速率限制：遵守Meta API的速率限制，并具备自动回退机制。
  • 回滚参考：每次更改都会被记录，以便用户随时撤销。
  • 安全层级：为每个账户提供沙盒、标准和生产访问级别。

“库”系统

服务器通过读取本地的Markdown文件（即用户的“库”）来获取客户智能信息。例如，结构可能包括：

your-vault/
    01_CLIENTS/my-client/
      00-profile.md        # 账户ID、像素、主页
      02-icp-personas.md   # 目标受众画像
      04-brand-voice.md    # 语气、语言、风格
      05-messaging-house.md # 营销角度、价值主张
      matrix.md

这种“库”系统确保了AI智能体在执行任务时，能够完全符合客户的品牌调性、目标受众特征及营销策略。
