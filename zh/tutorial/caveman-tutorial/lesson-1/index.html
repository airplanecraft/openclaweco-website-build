📖 核心概念讲解

1.1 冗余输出：AI Agent 的沉默杀手

当你用 AI 编程代理写代码时，你可能没注意到一个事实：Agent 说的话里，75% 是废话。

🗣️ 正常 Claude (69 tokens):
"The reason your React component is re-rendering is likely because
you're creating a new object reference on each render cycle. When you
pass an inline object as a prop, React's shallow comparison sees it
as a different object every time, which triggers a re-render. I'd
recommend using useMemo to memoize the object."

🪨 Caveman Claude (19 tokens):
"New object ref each render. Inline object prop = new ref = re-render.
Wrap in useMemo."

同样的技术结论，从 69 个 token 压缩到 19 个——省了 72%。

这不是偷工减料。被删掉的只有：

  • ❌ 客套话 ("The reason your...", "I'd recommend...")
  • ❌ 填充词 ("likely", "actually", "basically")
  • ❌ 重复解释 (同一个概念换三种说法)

保留下来的是：

  • ✅ 根因诊断 (new object ref)
  • ✅ 因果链 (inline prop → new ref → re-render)
  • ✅ 修复方案 (useMemo)

1.3 Caveman 的本质：行为规则注入器

很多人一听到 "安装 Caveman" 就以为是要配一个 MCP 服务器。不是的。

    subgraph Wrong["❌ 常见误解"]

    subgraph Right["✅ 真实机制"]

Caveman 的工作方式非常简单：

  1. 安装时，将一组 Prompt 规则和 Hook 脚本注册到你的 Agent 环境
  2. 会话启动时，Hook 自动将规则注入 Agent 的系统上下文
  3. Agent 回答时，按照注入的规则压缩输出
  4. 就这样。没有后台服务，没有网络请求，没有额外依赖

三个工具可以独立使用，也可以组合使用。本教程聚焦 caveman 本体。

1.5 科学背景：简短 ≠ 粗糙

你可能担心：压缩这么多，会不会丢失信息？

2026 年 3 月的论文 "Brevity Constraints Reverse Performance Hierarchies in Language Models" 给出了令人震惊的发现：

在某些基准测试上，强制要求模型简短回答反而提升了准确率 26 个百分点，并且完全逆转了不同模型间的性能排名。

这意味着冗长不仅浪费 token，还可能降低准确性。Caveman 的压缩本质上是在引导模型聚焦核心推理，而不是浪费算力在修辞上。
