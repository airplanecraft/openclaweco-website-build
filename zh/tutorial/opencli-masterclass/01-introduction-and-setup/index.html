第1期：OpenCLI 核心概念与环境搭建

更新于 2026/4/22

什么是 OpenCLI？

OpenCLI 是由开发者 jackwener 提供的一个可以把任何网站、Electron 客户端（如 Cursor、Notion）、本地命令行（gh、docker）转变为统一化命令行的工具的“大本营”（CLI Hub）。

最强大的是，它原生就是为 AI Agent 设计的。

为什么需要 OpenCLI？

大模型要使用工具，往往需要封装特定的 API，有时非常繁琐而且需要花费 API 的调用成本。如果能直接借用你当前浏览器已经登录好的状态来获取网页数据，是不是就成了零成本的抓取和操控工具？ OpenCLI 通过 Browser Bridge （浏览器桥接扩展） 直接利用你的 Chrome 登录态：

  1. 0 风险、0 门槛：不用折腾 cookie，无需复杂登录。
  2. 防检测（Anti-detection）：它本身修复了大量自动化控制容易被识别的特征，安全抓取。
  3. 零 Token 费用：运行时完全依靠本地驱动和网站原生接口，没有任何 LLM 消耗。

安装指南

安装 Node.js 后，通过 npm 全局安装：

npm install -g @jackwener/opencli

并配置浏览器扩展（从 GitHub Release 下载 opencli-extension.zip 并加载）。
