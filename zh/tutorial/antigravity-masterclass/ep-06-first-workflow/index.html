Phase 1 / Ep 06: 生成系统的第一条流水线 (Workflow)

⏱ 预计阅读 4 分钟 更新于 2026/4/22

你已经有了 GEMINI.md 的法律约束，有了 using-superpowers 的拦截机制，也用 planning-with-files 给系统扩展了无限脑存。

但如果你现在对 Agent 说：“把用户表建出来然后做个接口。” Agent 即便知道规矩，但依然会缺乏做事的“节奏感”——他可能依然会一次性丢个三千行带有报错的代码给你。

在正规互联网大厂，做任何事都有标准的 开发流水线 (SOP)。在 Agent 手里，这就叫 Workflow。

1. 工作流文件 .agents/workflows/feature-development.md

我们在项目中准备建立第一个标准化流程清单。把它存放到项目工作流目录下。这个文件是真正具有“指导行为执行力”的脚本。

---
description: T-Block 核心新功能开发标准化流程
---

# Feature Development Workflow (SOP)

当你接受到一个复杂的用户需求指令时，禁止立即写业务代码，请严格按先后顺序执行本 SOP：
// turbo-all

1. **意图获取与落地：** 
   读取需求本身，在心智里明确目的。第一时间调用并复习 `docs/task_plan.md`，如果不满足现有架构结构，立即启动研讨修改 Plan。

2. **行为防腐：**
   如果有任何对原项目架构不确定，优先发起搜索 (`grep`)，不要自己乱编文件路径。

3. **测试驱动 (TDD)：**
   只要确认了需要修改/新增代码功能，**必须**率先转往修改对应的 `.test.ts` / `.spec.ts` 单元测试跑一遍！测试必然抛出红色的报错（失败）。
   
4. **功能迭代：**
   以通过上述红字测测试为主旨，迭代增加业务代码。直到最后终端跑出绿色的 `Test Passed`。

5. **总结存档：**
   将遇到的困难或者有价值的设计理念，写入 `docs/findings.md`，再把你今天勾选的子任务更新在 `task_plan.md` 中。

2. "// turbo-all" 标记是什么黑魔法？

大家仔细看工作流里的这个 // turbo-all 注释！这在高级 Agent 系统里意义非凡。 通过这种声明标注，它直接授权底层 Agent：“在这个标准流程步骤下，如果需要运行一些安全探查终端命令或者非破坏性的目录查看，请放开手脚，自动加速自行判断，不需要每一步人类介入批准”。

这就构成了完全自治理状态最初的状态机。

3. 大功告成：Phase 1 验收

恭喜你！到这里为止，咱们的“地基”算是 100% 打牢固了。 总结一下现在的 T-Block 项目中存在的：

  • 一个简明的【One-Pager Intent】（代表人类意愿）
  • 【GEMINI.md】（规定整体底线与行为方式）
  • .agents/skills/using-superpowers/（防止幻觉的门神）
  • .agents/skills/planning-with-files/（防止脑损伤硬盘记录法）
  • .agents/workflows/feature-development.md（标准流水线上岗指南）

一个最完美的抗干扰赛博机械体诞生了。

准备好了吗？在接下来的 Phase 2 里，我们将按动点火开关。把人类的 One-Pager 扔进这部机器，在第一线感受什么叫由 AI主导的“极速 PRD 裂变”与“选型辩论”。

OpenCLI

OpenCLI是一款创新的AI原生运行时与统一命令行接口中心，旨在将任意网站、浏览器会话、Electron桌面应用及本地二进制工具标准化为确定性的命令行接口。它为人类用户和AI Agent提供了一个强大的自动化平台。用户可通过90+预构建适配器直接访问Bilibili、知乎等热门网站数据。AI Agent能利用`opencli browser`实时控制浏览器行为，或通过`explore`、`generate`等功能从现有网站行为中自动生成新的CLI适配器。OpenCLI还支持驱动Electron应用，并作为`gh`、`docker`等本地CLI工具的集成中心。其核心优势在于确保每次执行的确定性输出、复用浏览器登录状态保障账户安全，以及运行期间无LLM令牌消耗，并通过统一的`AGENT.md`集成赋能AI Agent无缝发现与执行工具。
